  ---
  What to Compute at Runtime (don't store)

  These should be derived from sp_sessions on the fly:

  - Streak — consecutive days with at least one session
  - Total time per subject — sum of durationMinutes filtered by subjectId
  - Daily/weekly progress % — sum of today's/this week's durationMinutes vs goal
  - Session count — .length of filtered sessions

  ---
  Key Design Decisions

  1. Soft deletes — archived: true on subjects instead of deleting, so session
  history is preserved
  2. Flat arrays — simple to serialize/deserialize, easy to filter/sort in JS
  3. Denormalized date — string YYYY-MM-DD on sessions makes day-grouping
  trivial without Date parsing
  4. perSubject goals are optional overrides — keeps the schema flexible; not
  every subject needs a target
  5. durationMinutes stored redundantly — avoids recalculating on every render
  loop

  JS Helper Functions (storage.js)

  These are the functions you'll write in storage.js — used by every page.

  // ── Generic ──────────────────────────────────────────────
  getAll(key)               // JSON.parse(localStorage.getItem(key)) || []
  saveAll(key, data)        // localStorage.setItem(key, JSON.stringify(data))
  generateId()              // crypto.randomUUID() short ID

  // ── Subjects ─────────────────────────────────────────────
  getSubjects()             // getAll('sp_subjects')
  saveSubject(subject)      // append or update by id
  deleteSubject(id)         // set archived: true (soft delete)

  // ── Sessions ─────────────────────────────────────────────
  getSessions()             // getAll('sp_sessions')
  saveSession(session)      // append new session
  getSessionsByDate(date)   // filter by session.date === 'YYYY-MM-DD'
  getSessionsBySubject(id)  // filter by session.subjectId
  getSessionsThisWeek()     // filter by date within current week range
  getTotalMinutes(sessions) // sum of durationMinutes

  // ── Assignments ──────────────────────────────────────────
  getAssignments()          // getAll('sp_assignments')
  saveAssignment(a)         // append or update
  toggleComplete(id)        // flip completed boolean
  getUpcoming(days)         // due within next N days, not completed
  getByDate(date)           // for calendar day view

  // ── Decks / Flashcards ───────────────────────────────────
  getDecks()                // getAll('sp_decks')
  saveDeck(deck)            // append or update deck (with cards inside)
  deleteDeck(id)            // remove deck by id
  addCard(deckId, card)     // push card into deck.cards
  updateCard(deckId, card)  // update card inside deck by card.id
  deleteCard(deckId, cardId)

  // ── Goals & Settings ─────────────────────────────────────
  getGoals()                // JSON.parse or default object
  saveGoals(goals)
  getSettings()             // JSON.parse or default object
  saveSettings(settings)

  // ── Stats (computed, not stored) ─────────────────────────
  getStreak()               // count consecutive days with sessions
  getDailyProgress()        // todayMinutes / goals.dailyTarget * 100
  getWeeklyProgress()       // weekMinutes / goals.weeklyTarget * 100
  getSubjectBreakdown()     // { subjectId: totalMinutes } for current week
